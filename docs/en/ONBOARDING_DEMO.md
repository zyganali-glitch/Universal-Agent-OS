# Onboarding and Demo Guide

Product name: Agent Governance OS Starter Kit

## Who This Is For
- teams building internal AI coding agents
- agencies standardizing repo bootstrap discipline
- founders who want repeatable agent governance across multiple products
- platform teams shipping a reusable agent starter layer

## 10-Minute Demo Flow
1. Open the package root.
2. Install a locale pack into a fresh repo root using `init-agent-os.ps1` (Windows) or `init-agent-os.sh` (macOS/Linux).
3. Read the installed locale `AGENTS.md` and global plan template.
4. Ask the agent to run a consultation-first startup flow.
5. Make the agent write or harden the repo-root `AGENT_OS_PLAN_TEMPLATE.md`.
6. Make the agent create a master roadmap plus child execution plans.
7. Confirm that implementation does not start before plan, evidence, and test strategy exist.
8. Show that manual locale-pack copying remains only a fallback path, not the primary operator experience.

## Onboarding Flow For Buyers
1. Choose locale and naming style.
2. Install the selected locale pack into the target repo with the provided installer.
3. Decide whether the target repo is single-language or bilingual.
4. Tailor the gate families for the domain.
5. Define the single-writer rule and supporting-role strategy.
6. Run the first project consultation session with the agent.
7. Create the repo-root global template and first active plan.
8. Validate that handoff, checkpoint, and archive behavior are present before coding begins.

## What To Show In A Live Demo
- The donor file explains the philosophy.
- The plan template shows operational rigor.
- The example repo shows how the package lands in a real project skeleton.
- The onboarding flow shows the package is usable, not just theoretical.

## Demo Success Criteria
- The buyer can identify the canonical donor file in under 30 seconds.
- The buyer can see how a new repo gets its own hardened root template.
- The buyer understands the main-agent and single-writer model.
- The buyer sees that roadmap-first planning happens before rapid prototyping.
